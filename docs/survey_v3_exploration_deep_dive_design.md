## 0. 문서 목적

본 설문을 **단순 시장조사**가 아니라, 아래 4가지를 동시에 검증하는 **탐색형(Exploratory) 검증 도구**로 확장한다.

| 검증 축 | 산출물 |
|---|---|
| **고객 문제** | Pain 페르소나 (Audit-heavy / CAPA-heavy / Trace / Delivery) |
| **타겟** | IT 성숙도·의사결정 구조·세그먼트 |
| **시스템 방향** | ERP-plug-in / MES-edge / Standalone QMS·SaaS / Excel-bridge / On-prem |
| **MVP 우선순위** | 유료 1순위 역량 · PoC KPI · WTP 교차 |

**설계 원칙** (탐색형 + 검증 엄밀성):

- **결론 유도 금지** — 솔루션 UVP(E섹션 intro) **이전**에 중립적 현황·선호(F·Deep Dive) 수집
- **깊이는 분기형** — 전원 동일 50문항이 아니라, 답변에 따라 **0 → 0-1 → 0-2**만 추가 노출
- **기존 v2 자산 유지** — B·C·D·E 핵심 진단·Van Westendorp·PoC 문항은 **재작성하지 않음**
- **구현 전 검증** — 본 문서는 `questions.ts` 반영 **전** 검토·승인용

## 2. Deep Dive 번호 체계

| 레벨 | 의미 | 노출 비율 (목표) |
|---|---|---|
| **0 (메인)** | 스크리닝·분류 — “무엇을 쓰는가 / 무엇이 아픈가” | 100% |
| **0-1** | 원인·맥락 — “왜 / 어디가 막히는가” | 40~60% |
| **0-2** | 행동·선호 — “무엇을 원하는가 / PoC KPI” | 15~25% |

**ID 규칙**: 부모 ID + 접미사. 예) `F0` → `F0-1` → `F0-2`, `C9` → `C9-1` → `C9-2`, `D2` → `D2-1`

**삽입 위치**:

- **신규 섹션 F** — D와 E **사이** (IT·시스템·MVP 중립 수집)
- **기존 문항 follow-up** — 해당 문항 **직후** (`showIf`)

**D2-1 · (0-1) 엑셀·수기를 유지하는 주된 이유**

**showIf**: D2 ∈ { 수기(1), 엑셀(2) }  
**유형**: 단일

- 현장·조직의 거부
- ERP/QMS 도입 실패 경험
- 규모상 충분
- 전사 IT 표준 없음
- 기타 (`hasText`)

### 5.5 D3 → D3-1 (절차서-실정 불일치)

**D3-1 · (0-1) 실정과 어긋난 **대표 공정/영역****

**showIf**: D3에 `2`(절차서가 실정과 안 맞음) 포함  
**유형**: 단일

- 수입검사 / 조립·가공 / 최종검사 / 출하 / 설계·4M변경 / 교육·자격 / 기타

### 5.6 A6 → A6-1 (타겟·의사결정)

**A6-1 · (0-1) 품질 IT·도구 도입 **예산·결정** 경험**

**showIf**: A6 ∈ { 최종 결정권, 직접 참여 }  
**유형**: 단일

- 100만 원 미만 자체 결정
- 100~500만 원 품의
- 500만 원 이상 경영 승인
- IT·구매 별도 프로세스
- 해당 경험 없음

### 5.7 E2 → E2-1 (장애요인 구체화)

**E2-1 · (0-1) 선택한 장애의 **구체적 우려****

**showIf**: E2 값별 분기

| E2 | E2-1 문항 |
|---|---|
| `1` 예산 | ROI 불명 / 경영 미관심 / CAPEX vs OPEX |
| `2` 현장 정착 | 추가 입력 / 교육 부담 / 기존 습관 |
| `3` ERP 중복 | **중복 시스템** — ERP / MES / QMS / 그룹웨어 (복수) |
| `4` 데이터 유출 | 고객 계약 / 공정기밀 / 망분리 / AI 학습 우려 |
| `5` 확신 부족 | 동종 사례 / 우리 공정 PoC / 내부 IT 승인 |

**showIf**: E2 = `6`(장애 없음), `7`(기타) → E2-1 **비노출**

### 5.8 E3 → E3-1 (PoC 전환 조건)

**E3-1 · (0-1) 유료 전환·본 도입 **조건****

**showIf**: E3 ∈ { 조건보고(2), 무료라면(3) }  
**유형**: 복수 (max 2)

- 업무시간 X% 이상 절감
- Audit 지적 N건 이상 감소
- CAPA 완결 기간 단축
- 경영진 승인 가능 수준의 ROI
- 월 OPEX Y만 원 이하
- 기타 (`hasText`)

## 6. showIf 구현 메모 (`questions.ts` 반영 시)

현 v2 패턴:

```typescript
showIf: { questionId: "D4_gate", value: "yes" }
showIf: { questionId: "B3A", exceptValues: ["6"] }
```

v3 확장 시 필요할 **로직 유형**:

| 유형 | 예시 |
|---|---|
| 단일 값 일치 | `F0-1b` ← F0-1에 `erp` 포함 |
| 복수 선택 개수 | `F0-2` ← F0-1 선택 ≥ 2 |
| 복수 선택 포함 | `D3-1` ← D3에 `2` 포함 |
| 1순위 분기 | `C9-1` ← C9의 rank-1 item id |
| 값 집합 | `F6-2` ← F6-1 ∈ { `poc`, `pilot` } |

> **구현 전**: `ShowIf` 타입 확장(contains, minSelections, rankFirst) 필요 여부를 `SurveyApp.tsx`와 함께 검토.
