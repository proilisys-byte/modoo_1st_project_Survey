### 1.2 Deep Dive 번호 규칙 (UI 동작)

```
[메인 문항 N] 보기를 클릭·선택
       ↓
해당 보기에 Deep Dive가 연결되어 있으면
       ↓
[N-1] 1단계 follow-up (맥락·원인)
[N-2] 2단계 follow-up (세부·결과) — 1-1 응답 후
[N-3] 3단계 follow-up (선택·구매·KPI) — 필요 시만
```

- **ID 규칙**: 화면·DB ID = `Q{N}`, Deep Dive = `Q{N}-1`, `Q{N}-2`, `Q{N}-3`
- **트리거**: 특정 **보기(value)** 선택 시에만 노출 — 전원 동일 Deep Dive 없음
- **필수 여부**: Deep Dive는 트리거된 경우에만 필수
