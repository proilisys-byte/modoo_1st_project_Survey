# 결과 페이지 점수 산출 기준 및 로직

> 이 문서는 설문 제출 후 나오는 **결과 페이지(우리 회사 ISO 실행력 진단 결과)** 의
> 점수·등급·리스크가 어떻게 계산되는지 설명합니다.
> 실제 계산 코드는 `lib/scoring.ts`, 문항 정의는 `lib/questions.ts`에 있습니다.
> 설계서 `PRO_ALI_SMART_설문조사_설계서.md`의 10절(스코어링 설계)을 코드로 구현한 것입니다.

---

## 1. 한눈에 보는 전체 흐름

```text
응답(answers)
  │
  ├─▶ ① 문항별 "잘함 정도(goodness)" 0~1 로 환산
  │
  ├─▶ ② 4개 영역별 점수 계산 (가중치 30/30/20/20)
  │
  ├─▶ ③ 총점 = 4개 영역 점수 합 (0~100)
  │
  ├─▶ ④ 총점 → 등급(A/B/C/D) + 개선 우선순위 문구
  │
  └─▶ ⑤ Pain 문항(C1~C8) → 리스크 TOP 3
```

---

## 2. 기본 개념: "잘함 정도(goodness)" 0~1

모든 문항을 **0(가장 나쁨) ~ 1(가장 좋음)** 사이의 값으로 바꾼 뒤 점수에 반영합니다.
문항 유형에 따라 환산 방식이 다릅니다.

### 2-1. 단일 선택 문항 → 정해진 매핑표 사용

응답 보기마다 "이 답이 얼마나 좋은 상태인가"를 미리 0~1로 정해 두었습니다
(코드의 `SINGLE_GOODNESS`).

| 문항 | 보기별 goodness |
|---|---|
| **B2** (문서-실제 불일치 지적) | ①있다·시정요구=0, ②구두지적=0.35, ③없다=1.0, ④모름=0.5 |
| **B4** (CAPA 완결 소요) | ①1주내=1.0, ②2~3주=0.7, ③1개월↑=0.4, ④효과확인 드묾=0.2, ⑤절차 미운영=0 |
| **B5** (교육으로 종결 비율) | ①10%미만=1.0, ②10~30%=0.75, ③30~60%=0.5, ④60~80%=0.25, ⑤80%↑=0, ⑥모름=0.4 |
| **B6** (유사 불량 재발) | ①없다=1.0, ②1~2회=0.6, ③3회↑=0.2, ④이력 없음=0 |
| **B3** (출하 클레임 건수) | ①없음=1.0, ②1~3건=0.75, ③4~10건=0.4, ④11건↑=0.15, ⑤집계 안함=0, ⑥기타=0.5 |
| **B7** (심사 준비 man-day) | ①3일미만=1.0, ②7일미만=0.7, ③12일미만=0.4, ④12일↑=0.15, ⑤모름=0.3, ⑥전담상주=0.2 |
| **D2** (문서관리 도구) | ①종이=0.1, ②엑셀·한글=0.3, ③그룹웨어·ERP 일부=0.6, ④전문 QMS=1.0, ⑤외부위탁=0.5 |

> 값이 낮을수록 "품질 리스크가 큰 응답"이라는 뜻입니다.
> 예: B3에서 "⑤ 집계하지 않아 모름"은 데이터 관리 부재 신호이므로 0점 처리합니다.

### 2-2. 빈도×영향도 문항(C1~C8) → 계산식으로 환산

C 섹션 각 문항은 **발생 빈도(1~5) × 업무 영향도(1~5) = Pain Score(1~25)** 를 먼저 구합니다.

```
Pain Score = 발생 빈도 × 업무 영향도        (최소 1, 최대 25)
goodness   = 1 − (Pain Score − 1) / 24      (Pain Score 1 → 1.0, 25 → 0.0)
```

즉 자주·심각하게 겪을수록 Pain Score가 커지고 goodness는 작아집니다.

| 예시 응답 | Pain Score | goodness |
|---|---|---|
| 빈도 1 × 영향 1 | 1 | 1.00 |
| 빈도 3 × 영향 3 | 9 | 0.67 |
| 빈도 5 × 영향 5 | 25 | 0.00 |

### 2-3. 무응답 처리

특정 문항에 응답이 없으면(또는 매핑에 없는 값이면) 그 문항은 **평균 계산에서 제외**합니다.
한 영역의 모든 문항이 비어 있으면 그 영역은 만점의 **50%(중립값)** 로 처리합니다.

## 3. 영역별 점수 (결과 페이지의 "영역별로 보는 우리 회사 현황")

총 100점을 4개 영역으로 나눠 배점합니다. 각 영역 점수는 다음과 같이 계산합니다.

```
영역 점수 = (그 영역에 속한 문항들의 goodness 평균) × 그 영역의 배점
```

| 영역 | 배점 | 포함 문항 | 의미 |
|---|---|---|---|
| **표준 실행력** | 30 | B2, C1, C2, C5 | 절차·표준이 현장에서 실제로 지켜지는가 |
| **부적합·CAPA 성숙도** | 30 | B4, B5, B6, C4, C6 | 불량이 났을 때 원인을 찾고 재발을 막는가 |
| **부서 협업·환류** | 20 | C3, C8 | 문제 정보가 부서 간에 공유·조치되는가 |
| **경영 가시성·데이터 관리** | 20 | B3, B7, C7, D2 | 품질 상태가 숫자·데이터로 관리·보고되는가 |

> 결과 화면의 막대는 `영역 점수 ÷ 배점` 비율로 채워집니다.
> 66% 이상 초록, 40~66% 노랑, 40% 미만 빨강으로 표시됩니다.

### 계산 예시 (표준 실행력, 배점 30)

- B2 = "③ 없다" → goodness 1.0
- C1 = 빈도3×영향4 = 12 → goodness 0.54
- C2 = 빈도2×영향2 = 4 → goodness 0.875
- C5 = 무응답 → 제외

평균 = (1.0 + 0.54 + 0.875) / 3 = 0.805
표준 실행력 점수 = 0.805 × 30 = **약 24점 / 30점**

## 4. 총점과 등급

```
총점 = 표준 실행력 + 부적합·CAPA 성숙도 + 부서 협업·환류 + 경영 가시성  (0~100, 반올림)
```

총점 구간에 따라 등급이 결정됩니다. 화면에는 **친근한 문구**를, 데이터에는 **등급 코드(A/B/C/D)** 와 정식 등급명을 함께 저장합니다.

| 총점 | 등급 코드 | 정식 등급명 (설계서 10.4) | 화면 표시 문구 |
|---|---|---|---|
| 80~100 | **A** | AI 운영혁신 준비 기업 | 잘 갖춰진 편이에요 |
| 60~79 | **B** | 실행 체계 보완 필요 기업 | 조금만 보완하면 돼요 |
| 40~59 | **C** | ISO 실행 갭 고위험 기업 | 실행이 서류를 못 따라가고 있어요 |
| 0~39 | **D** | 고객 Audit 리스크 집중관리 기업 | 지금 바로 챙겨야 해요 |

> 저장 컬럼: `grade`(화면 문구) · `grade_code`(A/B/C/D) · `grade_internal`(정식 등급명)

## 5. "개선 우선순위 제안"은 어떻게 나오나

결과 페이지 맨 아래 **"그래서, 무엇부터 하면 좋을까요?"** 문구는
**현재 등급(A/B/C/D)에 미리 연결된 고정 문구**입니다 (`GRADES[].plan`).

| 등급 | 개선 우선순위 제안 문구 |
|---|---|
| A | 품질 관리의 기본 틀이 잘 잡혀 있습니다. 이제 종이·엑셀 대신 데이터로 품질을 실시간 관리하는 단계로 넘어가 보시길 권합니다. |
| B | 기본은 갖췄지만 몇몇 영역이 약합니다. '영역별 현황'에서 막대가 가장 짧은 영역부터 4주 정도 집중해 개선해 보시길 권합니다. |
| C | 규정은 있지만 현장 실행이 따라가지 못하는 상태입니다. 매일 점검하는 체크시트와 불량 조치 이력 관리부터 시작하시길 권합니다. |
| D | 지금 상태로는 고객사 심사나 거래 유지에 문제가 생길 수 있습니다. 전문가 상담을 통해 급한 개선 항목부터 함께 정리해 보시길 권합니다. |

즉 지금은 **총점→등급→고정 문구**의 단순 매핑입니다.
응답자의 최저점 영역이나 리스크 TOP 3에 맞춰 문구를 개인화하는 로직은
아직 넣지 않았습니다(6절 참고).

## 6. "4주 플랜"이란? (현재 상태와 제안)

### 지금 코드에서의 의미

"4주 플랜"은 **B등급 제안 문구 안의 표현**일 뿐, 아직 시스템에 구체적인
주차별 실행 내용이 정의되어 있지 않습니다. 설계서 10.4절의
"취약 축(4개 축 중 최저점) 집중 개선 4주 플랜"이라는 표현을 문구로만 가져온 상태입니다.

> **정직한 현황:** 현재 결과 페이지는 "4주간 집중 개선을 권합니다"라고 안내만 하고,
> 실제로 무엇을 몇 주차에 하라는 구체 내용은 화면에 나오지 않습니다.

### 제안 — 최저점 영역 기반 4주 플랜 (미구현, 확정 시 반영 가능)

"영역별 점수 중 가장 낮은 영역"을 자동으로 골라, 그 영역에 맞는 4주 실행안을
보여주는 방식을 제안합니다. 예시안은 다음과 같습니다.

**표준 실행력이 최저점일 때**

| 주차 | 실행 내용 |
|---|---|
| 1주 | 납기 급할 때 자주 생략되는 절차 3개 선정 → 현장 인터뷰 |
| 2주 | 해당 절차를 현실에 맞게 간소화(1페이지 작업표준으로 축약) |
| 3주 | 4M 변경 발생 시 즉시 기록하는 간이 양식 도입 |
| 4주 | 표준 준수 여부 주간 점검 체크시트 운영 시작 |

**부적합·CAPA 성숙도가 최저점일 때**

| 주차 | 실행 내용 |
|---|---|
| 1주 | 최근 6개월 불량·재작업 이력 한 곳에 모으기 |
| 2주 | 반복 불량 TOP 3 선정 → 진짜 원인(4M) 분석 |
| 3주 | CAPA/8D 초안 템플릿 도입(원인분석 항목 필수화) |
| 4주 | 시정조치 효과확인 일정 등록·추적 시작 |

**부서 협업·환류가 최저점일 때**

| 주차 | 실행 내용 |
|---|---|
| 1주 | 출하검사 불합격 정보를 제조·개발에 공유하는 채널 개설 |
| 2주 | 주간 품질 공유 미팅(15분) 정례화 |
| 3주 | 고객 Audit 지적사항 조치 현황판(담당·기한) 운영 |
| 4주 | 미조치 항목 자동 알림 규칙 수립 |

**경영 가시성·데이터 관리가 최저점일 때**

| 주차 | 실행 내용 |
|---|---|
| 1주 | 불량·클레임·심사대응 시간을 숫자로 집계 시작 |
| 2주 | 품질 손실 비용(재작업·폐기·대응 인건비) 환산 기준 마련 |
| 3주 | 경영진용 1페이지 월간 품질 리포트 양식 확정 |
| 4주 | 첫 월간 리포트 작성·보고 |

> 이 4주 플랜을 실제 결과 페이지에 자동 노출하려면
> `lib/scoring.ts`에 "최저점 영역 → 4주 플랜" 매핑을 추가하면 됩니다.
> 원하시면 위 예시안을 코드에 반영해 드립니다.

## 7. 리스크 TOP 3

C1~C8 각 문항의 **Pain Score(빈도×영향도, 1~25)** 를 계산해,
값이 큰 순서대로 상위 3개를 뽑아 보여줍니다.
결과 화면의 "체감 부담 정도 X / 25점"이 이 Pain Score입니다.

| 문항 | 리스크 요약 |
|---|---|
| C1 | 바쁘면 정해진 순서를 건너뜁니다 |
| C2 | 사람이 바뀌면 품질도 흔들립니다 |
| C3 | 부서끼리 문제가 공유되지 않습니다 |
| C4 | 불량이 어디서 생겼는지 찾기 어렵습니다 |
| C5 | 바뀐 내용을 기록 없이 현장에서 먼저 씁니다 |
| C6 | 원인 분석보다 서류 마감에 급급합니다 |
| C7 | 품질 문제를 숫자로 보여줄 자료가 없습니다 |
| C8 | 지적받은 사항이 다음 심사까지 방치됩니다 |

> 주의 확인 문항(C_ATT)과 우선순위 강제 선택(C9)은 점수·리스크 계산에 쓰지 않습니다.
> C_ATT는 응답 품질 판정(빈도에서 2 선택 여부)에만, C9는 별도 분석용으로만 저장됩니다.

## 8. 점수에 반영되지 않는 문항

다음 문항은 **분석용으로 저장만** 되고 100점 스코어링에는 들어가지 않습니다.

- 섹션 A 전체(A1~A6): 세그먼트 분류용
- B1(심사 횟수), B3A(불량·재작업·폐기율)
- C9(우선순위 강제 선택), C_ATT(주의 확인)
- D1, D2A(혁신활동), D3, D4, D5
- 섹션 E 전체(수용도·가격): 리드·가격 분석용

> 이 문항들도 점수에 반영하고 싶으면 가중치를 정해 `diagnose()`에 추가하면 됩니다.
